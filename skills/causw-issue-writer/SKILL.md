# CAUSW Issue Writer

사용자가 제공한 정보와 `.github/ISSUE_TEMPLATE/`의 템플릿을 기반으로 GitHub 이슈 초안을 마크다운 파일로 생성한다.

## 이슈 타입 판별

사용자 요청에서 아래 키워드로 타입을 먼저 파악한다. 불명확하면 직접 질문한다.

| 타입 | 키워드 예시 | 라벨 | 제목 prefix |
|------|------------|------|-------------|
| **feature** | 기능, feature, API 개발, 구현, 추가 | `feat` | `[FEATURE]` |
| **bug** | 버그, bug, 오류, 에러, 안 됨, 500, 예외 | `bug` | `[BUG]` |
| **infra** | 인프라, 배포, CI/CD, 서버, DB, AWS, Nginx | `devops` | `[Infra]` |
| **general** | 그 외 전부, 범용, 일반, 논의, 리팩토링, 문서 | _(없음)_ | `[refactor]` 등 적절히 |

## 실행 순서

### 1단계: 이슈 타입 결정

- 요청 문맥에서 타입이 명확하면 바로 해당 템플릿으로 이동
- 불명확하면 한 줄로 질문: "어떤 종류의 이슈인가요? (feature / bug / infra / general)"

### 2단계: 사용자 정보 수집

사용자가 이미 설명한 내용은 재질문하지 말고 최대한 활용한다.
부족한 핵심 정보만 추가로 물어본다.

**feature** 이슈에서 필요한 정보:
- 개발할 기능 설명
- 관련 API Endpoint (있으면)
- 작업 목록 (체크리스트)

**bug** 이슈에서 필요한 정보:
- 버그 내용
- 발생 Endpoint 및 요청 정보
- 재현 절차 (알면)
- 기대 결과

**infra** 이슈에서 필요한 정보:
- 작업/이슈 내용
- 영향 범위 (EC2, RDS, S3, CI/CD 등)
- 작업 상세

**general** 이슈에서 필요한 정보:
- 한 줄 요약 제목
- 내용/배경
- 할 일 목록 (있으면)

### 3단계: 이슈 초안 작성

수집한 정보를 아래 해당 템플릿에 채워 마크다운 파일로 저장한다.

**파일명**: `.claude/out-docs/issue-draft-<타입>-<간략한-주제>.md`

디렉터리가 없으면 먼저 생성한다:
```bash
mkdir -p .claude/out-docs
```

#### Feature 템플릿

```markdown
---
title: "[FEATURE] <제목>"
labels: feat
---

## 📝 기능 설명

- <기능 상세 설명>

## 🚩 관련 API Endpoint

- <예: `POST /api/v1/users`>

## ✅ 개발 목표/작업 목록

- [ ] <작업1>
- [ ] <작업2>

## 🗒 참고 자료

- <링크, Swagger, RFC 등. 없으면 생략>
```

#### Bug 템플릿

```markdown

title: "[BUG] <제목>"

## 🧬 발생 API 및 요청 정보

- Endpoint: <예: `GET /api/v1/posts/1`>
- Request Body/Params: <예시 JSON 또는 파라미터>

## 🔄 재현 절차

1. <절차1>
2. <절차2>

## 📝 기대 결과

- <정상 동작 시 기대하는 결과>

## 🖼️ 에러 로그/스크린샷

- <에러 메시지, 로그. 없으면 생략>

## 💡 참고사항

- <DB 상태, 환경, 의심가는 코드 등. 없으면 생략>
```

## title prefix 선택 가이드 (general 타입)

사용자가 명시하지 않으면 아래 기준으로 적절한 prefix를 선택한다.

- `[REFACTOR]` — 리팩토링, 코드 정리
- `[DOCS]` — 문서, README, 주석
- `[TEST]` — 테스트 추가/수정
- `[CHORE]` — 설정, 빌드, 의존성
- `[DISCUSS]` — 논의, 의사결정
- `[Labels에 맞는 prefix]` — 그 외 자유롭게

## 완료 후

파일 경로를 사용자에게 알려주고, 작성된 이슈 제목과 주요 내용을 한 줄씩 요약해서 보여준다.

수정 요청이 있으면 파일을 바로 수정한다.

**GitHub에 바로 올리고 싶은지** 물어보고, 원하면 아래 명령어를 안내한다:

```bash
gh issue create \
  --title "<이슈 제목>" \
  --body-file <생성된-파일명>.md \
  --label "<라벨>"
```
